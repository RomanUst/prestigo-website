  <action>
The user must perform these steps in order. Claude cannot access Stripe Dashboard or Vercel Dashboard.

**Step 1: Generate HEALTH_SECRET locally**
Run this command and save the output:
```bash
openssl rand -hex 32
```
Save the output value — you will need it for Vercel and for the curl verification.

**Step 2: Register Stripe live-mode webhook**
1. Go to Stripe Dashboard — confirm the mode toggle at top shows **"Live"** (not "Test")
2. Navigate: Developers -> Workbench -> Webhooks tab
3. Click "Create an event destination"
4. Endpoint URL: `https://rideprestige.com/api/webhooks/stripe`
5. Events to listen for: select `payment_intent.succeeded` only
6. Save the endpoint
7. Open the endpoint detail page -> click "Signing secret" -> "Reveal secret"
8. Copy the `whsec_...` value — this is `STRIPE_WEBHOOK_SECRET`

**CRITICAL:** Confirm you are in **Live mode** before registering. A test-mode webhook will not receive live payment events. The signing secret from a test-mode endpoint will cause signature verification failures on live events.

**Step 3: Set env vars in Vercel**
1. Go to Vercel Dashboard -> rideprestige project -> Settings -> Environment Variables
2. Add `STRIPE_WEBHOOK_SECRET` = the `whsec_...` value from Step 2 — scope: **Production only**
3. Add `HEALTH_SECRET` = the hex string from Step 1 — scope: **Production only**
4. Confirm both variables appear in the list

**Step 4: Redeploy**
1. Go to Vercel Dashboard -> Deployments tab
2. Find the latest deployment -> click three-dot menu -> "Redeploy"
3. Wait for deployment to complete successfully

Vercel does NOT auto-pick up new env vars. The redeploy is mandatory.

Run the following curl command to verify the health endpoint is fully operational after Task 1 configuration. Replace TOKEN with the HEALTH_SECRET value generated in Task 1.

```bash
curl -s -H "Authorization: Bearer TOKEN" https://rideprestige.com/api/health | jq .
```

Expected response (HTTP 200):
```json
{
  "status": "ok",
  "services": {
    "supabase": { "ok": true },
    "stripe": { "ok": true },
    "resend": { "ok": true }
  }
}
```

**Troubleshooting if not all-green:**
- 401 `{"error":"Unauthorized"}`: HEALTH_SECRET env var is missing or Bearer token does not match. Check Vercel env vars and redeploy.
- `stripe.ok: false`: STRIPE_WEBHOOK_SECRET is missing or does not start with `whsec_`. Check Vercel env vars.
- `supabase.ok: false`: SUPABASE_URL or SUPABASE_SERVICE_ROLE_KEY may be wrong. These were set in Phase 7.
- `resend.ok: false`: RESEND_API_KEY may be invalid. Check Vercel env vars.
