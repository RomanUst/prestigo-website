   ```typescript
   function makeRequest(token?: string): Request {
     const headers: Record<string, string> = {}
     if (token) headers['authorization'] = `Bearer ${token}`
     return new Request('http://localhost/api/health', { method: 'GET', headers })
   }
   ```

6. Write 6 test cases in a `describe('/api/health')` block:

   **describe('Authorization'):**
   - "returns 401 when Authorization header is missing": call `GET(makeRequest())`, assert `res.status === 401`, assert `json.error === 'Unauthorized'`
   - "returns 401 when Bearer token is wrong": call `GET(makeRequest('wrong-token'))`, assert `res.status === 401`

   **describe('All services healthy'):**
   - "returns 200 with status ok when all probes pass": call `GET(makeRequest('test-secret-value'))`, assert `res.status === 200`, assert `json.status === 'ok'`, assert `json.services.supabase.ok === true`, `json.services.stripe.ok === true`, `json.services.resend.ok === true`

   **describe('Degraded states'):**
   - "returns 503 when Supabase probe fails": override supabaseStub's `from` to return `{ select: () => ({ limit: () => ({ error: { message: 'connection refused' } }) }) }`, call GET, assert `res.status === 503`, `json.status === 'degraded'`, `json.services.supabase.ok === false`
   - "returns 503 when Stripe keys are invalid": set `process.env.STRIPE_SECRET_KEY = 'invalid'` and `process.env.STRIPE_WEBHOOK_SECRET = 'invalid'`, call GET, assert `res.status === 503`, `json.services.stripe.ok === false`
   - "returns 503 when Resend probe fails": override `resendStub.domains.list` to return `{ error: { message: 'API key invalid' } }`, call GET, assert `res.status === 503`, `json.services.resend.ok === false`

Note on Supabase mock: The health route calls `supabase.from('bookings').select('id').limit(1)` which is a chained call. The mock must support this chain. Use:
```typescript
const supabaseStub = {
  from: vi.fn(() => ({
    select: vi.fn(() => ({
      limit: vi.fn(() => Promise.resolve({ error: null }))
    }))
  }))
}
```
Since `.limit(1)` returns a promise-like (Supabase uses builder pattern that resolves), the destructured `{ error }` comes from the awaited result. Make `limit` return `Promise.resolve({ error: null })` for the happy path.

IMPORTANT: After each degraded-state test, restore env vars and mocks in `beforeEach` using `vi.clearAllMocks()` and re-assigning env vars.
