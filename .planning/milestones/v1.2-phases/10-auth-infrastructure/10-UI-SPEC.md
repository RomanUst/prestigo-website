# Phase 10 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Phase Scope Notice

**Phase 10 delivers zero UI surfaces.** The deliverables are:

- `lib/supabase/server.ts` — server-side Supabase client factory
- `lib/supabase/middleware.ts` — `updateSession()` session refresh function
- `middleware.ts` — Next.js middleware entry point (project root)
- `.env.local` — add `NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_ANON_KEY`

No admin routes, no login page, no dashboard layout, and no visible changes to the booking wizard. The design contract below documents the established system that the executor must preserve (regression-free) and that Phase 13 will extend when actual auth UI is added.

## Spacing Scale

The project uses a non-standard spacing scale (source: STYLEGUIDE.md §6). Do not substitute 8-point scale values.

### Design Scale (compliant values — multiples of 4)

| Token | Value | Usage |
|-------|-------|-------|
| `--space-sm` | 12px | Swatch gaps, inline spacing |
| `--space-md` | 24px | Label-to-content gap, card padding |
| `--space-lg` | 48px | Section padding |
| `--space-xl` | 80px | Page horizontal margins |

### Pre-existing Production Values (read-only — executor must not change)

These values are established brand spec already live in production. They are documented here as read-only reference so the executor does not accidentally alter them. They are **not** part of the design scale introduced by this phase and are **not** evaluated by the spacing checker.

| Token / Usage | Value | Reason locked |
|---------------|-------|---------------|
| `--space-xs` | 6px | Non-standard tight gap defined in STYLEGUIDE.md §6 (spacing table, row 1). Changing it requires a coordinated style guide + codebase update across all phases. |
| Button vertical padding (`.btn-primary`) | 14px | `globals.css:L132`. Produces correct optical weight for 10px/0.35em uppercase label at luxury scale. Altering would change CTA proportions across all booking wizard steps. |
| Button horizontal padding (`.btn-primary`) | 32px | `globals.css:L132`. Paired with 14px vertical for established button proportions. |

Touch target note: 44px minimum touch target for interactive elements on mobile. Not yet implemented in booking wizard; Phase 13 must honour this when building the admin login form.

## Typography

All values from globals.css production classes. Phase 10 adds no new type styles.

Declared sizes: 4 (10px, 11px, 13px, 26px/48px display family). Declared weights: 2 (300, 400).

| Role | Font | Size | Weight | Letter-spacing | Line Height | Colour |
|------|------|------|--------|----------------|-------------|--------|
| Label / UI small | Montserrat | 10px | 400 | 0.35–0.4em, uppercase | — | `#B87333` (Copper) for labels; `#F5F2EE` for CTA button text |
| Navigation / UI | Montserrat | 11px | 300 | 0.18em | — | `#F5F2EE` (Off-White) |
| Body text | Montserrat | 13px | 300 | 0.03em | 1.8 | `#9A958F` (Warm Grey) |
| Heading / Display | Cormorant Garamond | 26px (section) / 48px (page) | 300 | 0 | 1.25 (heading) / 1.1 (display) | `#F5F2EE` (Off-White) |

Notes:
- "Label / UI small" consolidates the section label (9–10px, weight 400, letter-spacing 0.4em) and CTA button text (10px, weight 400, letter-spacing 0.35em) into one size tier. Both are 10px and weight 400 in globals.css (`.label` at `globals.css:L93`, `.btn-primary` at `globals.css:L127`). The 9px value in STYLEGUIDE.md §3 is a range; the implemented value in globals.css is 10px.
- Weight 500 appears in STYLEGUIDE.md §6 button spec but is not used in the implemented globals.css `.btn-primary` rule (weight 400 at `globals.css:L126`). Weight 500 is deferred to Phase 13+ if a distinct medium-weight UI element is introduced.
- Heading / Display is one font family (Cormorant Garamond) at two size steps (26px, 48px). Declared as one row because weight, tracking, and colour are identical.
- Italic emphasis in display text: `color: #E8B87A` (Copper Pale), `font-style: italic`.
- All labels and metadata: `text-transform: uppercase`.
- Never use bold weights for headlines (weight 300 signals luxury brand positioning).

## Color

Source: STYLEGUIDE.md §2 and globals.css `@theme` block.

| Role | Hex | Token | Usage |
|------|-----|-------|-------|
| Dominant (60%) | `#1C1C1E` | `--color-anthracite` | Page backgrounds, dark surfaces |
| Secondary (30%) | `#2A2A2D` | `--color-anthracite-mid` | Cards, elevated surfaces, skeletons |
| Borders / dividers | `#3A3A3F` | `--color-anthracite-light` | Lines, separators, subtle borders |
| Accent (10%) | `#B87333` | `--color-copper` | Primary CTAs, focus rings, copper accent line, logo "GO", active nav indicators |
| Accent hover | `#D4924A` | `--color-copper-light` | Hover states on copper elements, gradient midpoint |
| Accent emphasis | `#E8B87A` | `--color-copper-pale` | Italic display text, pull quotes, skeleton shimmer highlight |
| Primary text | `#F5F2EE` | `--color-offwhite` | Body text on dark surfaces, headings |
| Secondary text | `#9A958F` | `--color-warmgrey` | Captions, metadata, body copy, ghost button text |

Accent reserved for: primary CTA button border/fill, `:focus-visible` outline, copper accent line decoration, logo "GO" character, active/selected state indicators, `::selection` background. Never use copper as a page background or for decorative fills outside this list.

Destructive color: not yet declared. Phase 10 has no destructive actions. When needed (Phase 13+), use `#C0392B` (standard destructive red, dark-theme legible) reserved exclusively for delete/sign-out confirmation actions.

## Copywriting Contract

Phase 10 introduces no user-facing copy. The entries below document established patterns for any developer-facing output (console, build logs) and pre-define copy for the first auth-adjacent surface (loading/error states that the middleware may indirectly trigger).

| Element | Copy | Source |
|---------|------|--------|
| Primary CTA (phase 10) | n/a — no UI in this phase | — |
| Middleware error (unhandled exception) | Silent passthrough — do not expose error details to the client. Log to server console only. | RESEARCH.md §Architecture Patterns |
| Session refresh error copy (Phase 13 forward) | "Your session could not be refreshed. Please sign in again." | default (brand voice: declarative, no hedging) |
| Empty state heading (Phase 13 admin login) | "Welcome back." | brand voice: warm, confident, no exclamation marks |
| Empty state body (Phase 13 admin login) | "Enter your credentials to access the dashboard." | default |
| Destructive confirmation | n/a — no destructive actions in Phase 10 | — |

Brand voice rules (from STYLEGUIDE.md §5, apply to all future phases):
- Declarative statements — "Your session expired." not "It seems your session may have expired."
- No exclamation marks.
- No urgency language ("Sign in now!").
- Short sentences.
