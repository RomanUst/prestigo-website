Create `prestigo/app/admin/(dashboard)/bookings/page.tsx` as a `'use client'` component.

**NOTE on 'use client':** RESEARCH.md marks this as a "Server Component shell", but the bookings page requires `'use client'` because it manages client-side KPI state (fetching today's count and this week's revenue on mount, updating state). Unlike the pricing page (which fetches server-side and passes to a client form), this page's KPI data fetching is interleaved with the client-side BookingsTable lifecycle. A Server Component version would require a separate KPI API endpoint or parallel server-side fetch — unnecessary complexity for 2 lightweight summary queries.

```tsx
'use client'
import { useState, useEffect } from 'react'
import { KPICard } from '@/components/admin/KPICard'
import BookingsTable from '@/components/admin/BookingsTable'
```

**KPI data fetching:**
On mount, fetch today's bookings count and this week's revenue from the bookings API:
1. Today's count: `GET /api/admin/bookings?startDate=${todayISO}&endDate=${todayISO}&limit=1` — use `total` from response
2. This week's revenue: `GET /api/admin/bookings?startDate=${mondayISO}&endDate=${sundayISO}&limit=100` — sum `amount_czk` from response bookings

Helper: `getMonday(date)` returns the Monday of the current week in ISO format. `getSunday(date)` returns Sunday.

**Page layout per spec:**
```
[h1: "Bookings"] marginBottom: 16px
[KPI summary row: flex, gap: 16px] marginBottom: 24px
  [KPICard label="TODAY" value="{count}" subLabel="bookings"]
  [KPICard label="THIS WEEK" value="{sum} CZK" subLabel="CZK revenue"]  (note: format number with space as thousands separator)
[BookingsTable]
```

**Number formatting:** Use `new Intl.NumberFormat('cs-CZ').format(value)` for CZK values with space as thousands separator (e.g., "42 800").

**h1 style:** Same as all admin pages: `fontFamily: 'var(--font-cormorant)'`, `fontSize: '28px'`, `fontWeight: 400`, `color: 'var(--offwhite)'`, `letterSpacing: '0.08em'`, `marginBottom: '16px'`
