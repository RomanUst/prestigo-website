## Critical Gap: pricing_globals Not Consumed by Booking Wizard

### What was built
- `pricing_globals` table: stores airport_fee, night_coefficient, holiday_coefficient, extra_child_seat, extra_meet_greet, extra_luggage
- Phase 14 PUT `/api/admin/pricing`: correctly upserts both `pricing_config` (rates) and `pricing_globals` (extras/coefficients)
- Phase 14 also calls `revalidateTag('pricing-config')` to bust the cached rates
- Phase 16 PricingForm: correctly renders and saves all fields from both tables

### What's missing
- `lib/pricing-config.ts` `getPricingConfig()` only queries `pricing_config` (per-class rates). It never touches `pricing_globals`.
- `/api/calculate-price/route.ts` calls `getPricingConfig()` and gets `{ ratePerKm, hourlyRate, dailyRate }`. It never has access to airport_fee, night/holiday coefficients, or extra fees from the database.
- The booking wizard's price calculation uses hardcoded fallback values for these parameters (or zeros).

### Impact
Operator edits airport fee or night coefficient → admin UI shows "Pricing saved." → DB updated correctly → booking wizard unchanged.

### Fix required
`getPricingConfig()` must be extended to also query `pricing_globals` and return the globals alongside rates. The `calculate-price` route must use these DB values instead of hardcoded values.
