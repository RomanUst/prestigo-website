# Phase 1 — UI Design Contract: Foundation & Trip Entry

> Visual and interaction contract for the /book page wizard shell and Step 1 (trip type + route entry). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — custom CSS via `globals.css` |
| Preset | not applicable |
| Component library | none — hand-built components using CSS custom properties |
| Icon library | Lucide React (for swap icon, plane icon, clear ×, stepper ±) |
| Font — Display | Cormorant Garamond (300 Light, italic variant) |
| Font — Body / UI | Montserrat (300 Light, 400 Regular) |

**Rationale (source: STYLEGUIDE.md + globals.css):** The project uses CSS custom properties (`var(--copper)`, `var(--anthracite)`, etc.) declared in `globals.css`. No Tailwind color utilities — brand tokens only. No shadcn. All new components must follow this pattern.

---

## Spacing Scale

Declared values (project uses a hybrid: style guide tokens + Tailwind multiples-of-4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap, bullet dots, indicator dots |
| sm | 8px | Stepper button internal padding, tag gaps |
| md | 16px | Field label-to-input gap, inline element spacing |
| lg | 24px | Between form fields (vertical stack), progress step gap |
| xl | 32px | Wizard inner padding (horizontal), button row top margin |
| 2xl | 48px | Section top/bottom padding (mobile wizard body) |
| 3xl | 64px | Page-level top padding below Nav on desktop |

Exceptions:
- Page header top padding: `pt-32` (~128px) desktop, `pt-40` (~160px) — matches existing `/book` page header (source: `book/page.tsx`)
- Content container: `max-w-7xl mx-auto px-6 md:px-12` — must match site-wide container (source: CONTEXT.md)
- Touch targets (mobile): minimum 44px height for all interactive elements (stepper buttons, tab pills, Next/Back buttons)
- Progress bar circle size: 32px diameter

## Typography

All sizes and weights extracted from `globals.css` and `STYLEGUIDE.md`. No new type styles introduced in this phase.

| Role | Font | Size | Weight | Line Height | Letter Spacing | Color |
|------|------|------|--------|-------------|----------------|-------|
| Step heading ("Select your journey") | Cormorant Garamond | 28px | 300 | 1.25 | 0 | `var(--offwhite)` |
| Field label / Tab / UI label / Button label / Progress step number | Montserrat | 9px | 400 | 1 | 0.35em | context-dependent — see notes below |
| Body / helper text / Autocomplete suggestion | Montserrat | 13px | 300 | 1.8 | 0.03em | `var(--warmgrey)` — use `.body-text` class; matched substring weight 400, color `var(--copper)` |
| Input value text | Montserrat | 14px | 300 | 1.5 | 0.02em | `var(--offwhite)` |

Notes on 9px usage:
- Field label: letter-spacing 0.4em, `var(--copper)` — use `.label` class
- Tab UI label: letter-spacing 0.35em, `var(--offwhite)` active / `var(--warmgrey)` inactive
- Button label (Next / Back): letter-spacing 0.35em, per `.btn-primary` / `.btn-ghost`
- Progress step number: letter-spacing 0.1em, `var(--anthracite)` on active circle / `var(--warmgrey)` on pending
- Duration segment label: letter-spacing 0.2em, `var(--offwhite)` active / `var(--warmgrey)` inactive
- Place type sub-label (e.g., "Airport") in autocomplete: letter-spacing 0.2em, `var(--warmgrey)`, uppercase, right-aligned

Rules (source: STYLEGUIDE.md):
- Never use bold (600+) for headings — lightness signals luxury
- Italic emphasis in display text uses `var(--copper-pale)` (#E8B87A)
- All labels and tabs: `text-transform: uppercase`

## Color

Source: `globals.css` CSS custom properties + STYLEGUIDE.md.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#1C1C1E` (`var(--anthracite)`) | Full-bleed wizard background, page body |
| Secondary (30%) | `#2A2A2D` (`var(--anthracite-mid)`) | Autocomplete dropdown background, progress bar track background |
| Surface / borders | `#3A3A3F` (`var(--anthracite-light)`) | Field borders, dividers, progress bar line, pending step circles |
| Accent (10%) | `#B87333` (`var(--copper)`) | See reserved-for list below |
| Accent hover | `#D4924A` (`var(--copper-light)`) | Copper button fill on hover, autocomplete match highlight on hover row |
| Accent pale | `#E8B87A` (`var(--copper-pale)`) | Italic display emphasis only |
| Primary text | `#F5F2EE` (`var(--offwhite)`) | Input values, step heading, active tab text, progress number on active circle |
| Secondary text | `#9A958F` (`var(--warmgrey)`) | Helper text, inactive tab text, Back button, stepper count number |
| Destructive | `#C0392B` (standard red — not in current tokens) | Field validation error border and error message text |

**Accent (`var(--copper)`) reserved exclusively for:**
1. Active trip type tab — underline or text (not filled background)
2. Progress bar — active step circle fill and completed step tick
3. `.copper-line` decorative accent (step header separator)
4. Field label text (`.label` class)
5. Stepper `+` / `−` button text
6. Read-only airport field — icon color and text color
7. Focus outline on all interactive elements (`:focus-visible` — already in globals.css)
8. Autocomplete matched substring highlighting
9. `.btn-primary` border (unfilled state) and fill (hover/active state)
10. Scrollbar thumb on hover

**Copper must NOT appear as:** general background fill, tab active background, card background, input background.

## Component Inventory

Components to build in this phase, with their visual specifications:

### ProgressBar

- 6 numbered circles connected by a horizontal line
- Circle diameter: 32px
- Active circle: filled `var(--copper)`, number `var(--anthracite)`, weight 400
- Completed circle: filled `var(--copper)`, checkmark SVG (16px) in `var(--anthracite)`
- Pending circle: border `1px solid var(--anthracite-light)`, background `var(--anthracite-mid)`, number `var(--warmgrey)`
- Connecting line: 1px, `var(--anthracite-light)`, horizontal, centered between circles
- Completed segment of line: `var(--copper)`
- Bar sits below `<Nav />`, above wizard form
- Spacing: 24px above and below the bar

### Trip Type Tab Row

- 5 pills in a horizontal scrollable row: ONE-WAY | AIRPORT PICKUP | AIRPORT DROPOFF | HOURLY | DAILY
- Font: Montserrat 9px, weight 400, letter-spacing 0.35em, uppercase
- Active tab: copper bottom border (2px) OR copper text — not filled background
- Inactive tab: `var(--warmgrey)` text, no border
- Hover: `var(--offwhite)` text
- Tab row: sticky within the wizard form header so it stays visible on scroll
- Bottom border separator under entire row: `1px solid var(--anthracite-light)`
- Padding: 12px top and bottom per tab, 16px horizontal per tab

### AddressInput (Google Places Autocomplete)

- Input field styling: background `var(--anthracite-mid)`, border `1px solid var(--anthracite-light)`, padding `12px 16px`
- Input text: Montserrat 14px, weight 300, `var(--offwhite)`
- Placeholder: Montserrat 13px, weight 300, `var(--warmgrey)` at 60% opacity
- Focus state: border-color `var(--copper)`, copper `:focus-visible` outline (already global)
- Clear (×) button: appears when field has value, `var(--warmgrey)`, right-aligned inside field, 16px tap zone minimum
- Autocomplete dropdown: background `var(--anthracite-mid)`, `1px solid var(--anthracite-light)` border, `z-index: 50`
- Suggestion row: 44px minimum height, 16px horizontal padding
- Suggestion row hover: background `var(--anthracite-light)`
- Matched substring: `var(--copper)`, weight 400
- Non-matched text: `var(--warmgrey)`, weight 300
- Place type sub-label (e.g., "Airport"): 9px, `var(--warmgrey)`, uppercase, letter-spacing 0.2em, shown right-aligned
- Debounce: 300ms before calling Places API (source: CONTEXT.md)
- Minimum 2 characters before suggestions show (source: CONTEXT.md)

### Read-Only Airport Field

- Visual: same field dimensions as AddressInput
- Background: `var(--anthracite)` (slightly dimmer than editable fields — uses dominant bg)
- Border: `1px solid var(--anthracite-light)` with `opacity: 0.6`
- Text: `var(--copper)` — signals it is auto-set
- Leading icon: plane SVG (16px, `var(--copper)`), left-aligned inside field
- Cursor: `default` — field is not `disabled` but is read-only (source: CONTEXT.md)
- No clear (×) button on read-only field

### Swap Icon (between Origin / Destination on one-way)

- Position: centered horizontally between the two address fields, vertically centered
- Icon: `ArrowUpDown` from Lucide React, 16px, `var(--warmgrey)`
- Hover: `var(--copper)`
- Tap target: 32px × 32px minimum
- Action: swaps origin ↔ destination values in Zustand store
- `aria-label="Swap origin and destination"` on the button element

### Stepper (Passengers / Luggage)

- Layout: `[−] [count] [+]` inline row
- `−` / `+` buttons: 32px × 32px, text `var(--copper)`, border `1px solid var(--anthracite-light)`
- Count: Montserrat 14px, weight 300, `var(--warmgrey)`, min-width 32px, centered
- Disabled state of `−`: opacity 0.3 when at minimum (1 for passengers, 0 for luggage)
- Disabled state of `+`: opacity 0.3 when at maximum (8)
- Both steppers inline on one row on desktop (24px gap between them), stacked on mobile

### Duration Selector (Hourly only)

- Segmented button group: `1h | 2h | 3h | 4h | 6h | 8h | 12h`
- Font: Montserrat 9px, weight 400, letter-spacing 0.2em
- Each segment: 44px minimum height, equal width
- Active: `var(--copper)` border bottom 2px, `var(--offwhite)` text
- Inactive: `var(--warmgrey)` text, `var(--anthracite-light)` border on all sides
- Group has outer border `1px solid var(--anthracite-light)`, no gap between segments

### Next Button

- Uses `.btn-primary` class exactly — no overrides
- Spec: copper border (1px), transparent background, `var(--offwhite)` text, fills copper on hover
- Disabled state: `opacity: 0.4`, `cursor: not-allowed`, hover has no effect
- Mobile: fixed at bottom of viewport, full width, `padding: 18px`
- Desktop: inline below form fields, right-aligned, standard padding `14px 32px`
- Label: "Continue" (see Copywriting Contract)

### Back Button

- Uses `.btn-ghost` class exactly — no overrides
- Mobile: fixed at bottom of viewport, left of Next button
- Desktop: inline to the left of Next button
- Label: "Back" (only shown on steps 2–6; hidden on Step 1)

### Step Entry Animation

- Use CSS `fadeUp` keyframe from globals.css: `opacity: 0 → 1, translateY(24px) → 0`
- Duration: 0.3s (shorter than global 0.9s — form elements not page sections)
- Easing: `cubic-bezier(0.22, 1, 0.36, 1)`
- No framer-motion in Phase 1 (source: CONTEXT.md — prefer CSS to avoid dep)
- Respects `prefers-reduced-motion` (already handled globally in globals.css)

## Layout Contract

### /book Page Structure

```
<Nav />                          ← keep, do not modify
<section> page header            ← keep existing header (label, copper-line, h1, body-text)
<BookingWizard>
  <ProgressBar />                ← sits at top of wizard
  <div> tab row (sticky)         ← trip type tabs, sticky within form scroll
  <form> step content            ← step fields
  <div> button row               ← Back + Next
</BookingWizard>
<Footer />                       ← keep, do not modify
```

### Wizard Width

- Container: `max-w-7xl mx-auto px-6 md:px-12` (matches site-wide, source: CONTEXT.md)
- Form column: `max-w-xl` (600px) — single-column, centered or left-aligned within container
- Steps 2–6 stubs: same wrapper, content is a placeholder message in `.body-text` style

### Mobile Layout

- Wizard fills viewport height minus Nav height
- Step content is scrollable
- Next / Back buttons fixed at bottom: `position: fixed; bottom: 0; left: 0; right: 0`
- Fixed button bar background: `var(--anthracite)` with `border-top: 1px solid var(--anthracite-light)`
- Button bar height: 72px, buttons full-width within 16px horizontal padding

### Desktop Layout

- No fixed button bar — buttons are inline below fields
- Step content not fullscreen — wizard is a contained section below the page header

## Interaction States

### Field Validation (on Next attempt with empty required fields)

- Trigger: user clicks Next with one or more required fields empty
- Empty required field: border changes to `#C0392B` (destructive red), 1px solid
- Error message below field: Montserrat 13px, weight 400, color `#C0392B`, margin-top 8px
- Error clears when field receives a valid value (on change, not on blur)
- No toast notifications — all errors are inline

### Address Field States

| State | Border | Background | Text |
|-------|--------|------------|------|
| Default | `var(--anthracite-light)` | `var(--anthracite-mid)` | placeholder warmgrey |
| Focus | `var(--copper)` | `var(--anthracite-mid)` | `var(--offwhite)` |
| Filled | `var(--anthracite-light)` | `var(--anthracite-mid)` | `var(--offwhite)` |
| Error (empty on Next) | `#C0392B` | `var(--anthracite-mid)` | `var(--offwhite)` |
| Read-only (airport) | `var(--anthracite-light)` at 0.6 opacity | `var(--anthracite)` | `var(--copper)` |

### Next Button Enabled Condition

Step 1 Next becomes enabled when ALL of the following are true:
- `tripType` is set (always true after initial render — default to `transfer`)
- `origin` has a selected Places result (not just typed text)
- `destination` has a selected Places result — OR — `tripType === 'hourly'` and `hours` is set
- `passengers` ≥ 1

## Accessibility Requirements

Phase 1 scoped requirements (full accessibility pass is Phase 6, source: UX-04, UX-05):

- All interactive elements have `aria-label` (source: UX-04)
- Trip type tab group: `role="tablist"`, each tab `role="tab"`, `aria-selected`
- Progress bar: `aria-label="Booking progress: Step N of 6"`, `aria-current="step"` on active circle
- Address inputs: `aria-label` matching field label, `aria-autocomplete="list"`, `aria-expanded` on dropdown open
- Autocomplete listbox: `role="listbox"`, each suggestion `role="option"`
- Stepper buttons: `aria-label="Increase passengers"` / `aria-label="Decrease luggage"` etc.
- Swap icon button: `aria-label="Swap origin and destination"`
- Next button: `aria-disabled="true"` when disabled (not `disabled` attribute alone)
- `:focus-visible` copper outline is already globally set — do not override
- Keyboard: tab order flows top-to-bottom through all visible fields, then Next button
