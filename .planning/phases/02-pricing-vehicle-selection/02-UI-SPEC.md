the booking wizard follows standard 4-pt multiples as used in Phase 1 Tailwind classes.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, tight inline padding |
| sm | 8px | Between card meta items (passenger icon + count) |
| md | 16px | Default element spacing, input padding |
| lg | 24px | Card internal padding, section padding within steps |
| xl | 32px | Gap between vehicle cards (desktop row), button padding horizontal |
| 2xl | 48px | Gap between Step 2 calendar and time-slot list |

- Touch targets on mobile time-slot rows: minimum 44px height (accessibility — UX-01)

- PriceSummary mobile fixed bottom bar: 56px height (above nav bar safe area)

- All labels: `text-transform: uppercase`. No exceptions.

## Color

All values sourced from `globals.css` `:root` custom properties. Use CSS vars, not hardcoded hex.

| Role | CSS Variable | Hex | Usage |
|------|-------------|-----|-------|
| Dominant (60%) | `var(--anthracite)` | `#1C1C1E` | Page background, step background |
| Secondary (30%) | `var(--anthracite-mid)` | `#2A2A2D` | Vehicle cards, PriceSummary panel, calendar grid background |
| Accent (10%) | `var(--copper)` | `#B87333` | Reserved elements listed below |
| Destructive | n/a | — | No destructive actions in Phase 2 |

Accent (`var(--copper)`) is reserved exclusively for:
1. Selected vehicle card border ring (2px solid copper)
2. Selected date cell fill in calendar (copper background, anthracite text)
3. Focus-visible outline on all interactive elements (`:focus-visible` — already in globals.css)
4. `.label` class text (field labels, section labels)
5. `.btn-primary` border and hover fill
6. Skeleton shimmer highlight color (copper-pale `#E8B87A` at low opacity, not full copper)

Accent must NOT be used for:
- Unselected card borders (use `var(--anthracite-light)`)
- Hover background on unselected cards (use subtle `var(--anthracite-light)` lift)
- Time slot highlight text (use `var(--offwhite)` on anthracite-mid background)

Secondary semantic colors:
- `var(--warmgrey)` — secondary text, captions, disabled state text
- `var(--anthracite-light)` — borders, dividers, unselected card outlines
- `var(--copper-pale)` — skeleton shimmer gradient endpoint only

## Component Specifications

### DatePicker (Step 2)

- Library: react-day-picker (~5KB). Source: 02-CONTEXT.md decisions.
- Layout: Inline calendar, always visible — not a popover, not a drawer.
- Calendar grid: 7-column week grid. Day cell: 44px × 44px on mobile, 40px × 40px on desktop.
- Past dates: `color: var(--warmgrey)`, `cursor: not-allowed`, `pointer-events: none`.
- Today: subtle `var(--anthracite-light)` ring (1px).
- Selected date: `background: var(--copper)`, `color: var(--anthracite)`, no border-radius (flat aesthetic consistent with brand).
- Hover on valid dates: `background: var(--anthracite-mid)`, `color: var(--offwhite)`.
- Month navigation arrows: `.btn-ghost` style — `var(--warmgrey)` color, `var(--anthracite-light)` border.
- For Daily Hire: second calendar row for return date appears below outbound calendar with a `.label` "RETURN DATE" separator. Same styling rules apply.

### TimePicker (Step 2)

- Implementation: scrollable list of 15-minute slots (00:00–23:45).
- Each row: 44px min-height, `padding: 0 16px`, `font-size: 13px`, Montserrat 300, `color: var(--warmgrey)`.
- Selected slot: `background: var(--anthracite-mid)`, `color: var(--offwhite)`, left copper bar `4px solid var(--copper)`.
- Hover: `background: var(--anthracite-mid)`, `color: var(--offwhite)`.
- Container: `max-height: 240px`, `overflow-y: auto`, thin scrollbar (matches `::-webkit-scrollbar` in globals.css).
- No free-text entry. No native `<input type="time">`.

### Vehicle Card (Step 3)

- Layout: 3 cards in a CSS grid row on desktop (`grid-cols-3`, gap `xl` 32px). Stacked full-width on mobile (`grid-cols-1`, gap `lg` 24px). No horizontal carousel.
- Card background: `var(--anthracite-mid)`.
- Card border default: `1px solid var(--anthracite-light)`.
- Card border selected: `2px solid var(--copper)` + background lifts to `var(--anthracite-light)`.
- Card border-radius: 4px (matches STYLEGUIDE).
- Card internal padding: `24px` all sides.
- Card content order (top to bottom):
  1. Vehicle photo — full-width, `aspect-ratio: 16/9`, `object-fit: cover`.
  2. Class name — 13px / Montserrat 400 / offwhite / `text-transform: uppercase` / `letter-spacing: 0.25em`.
  3. Capacity row — passenger icon + count + luggage icon + count, 13px / warmgrey.
  4. Price — 20px / Montserrat 500 / offwhite. Or "Request a quote" — 13px / 300 / warmgrey.
- No amenities list on card (deferred — 02-CONTEXT.md deferred section).
- Transition on selection: `border-color 0.2s ease, background-color 0.2s ease`.

### Price Loading Skeleton

- Location: occupies the price slot within the vehicle card.
- Implementation: CSS `@keyframes shimmer` (copper-pale to anthracite-mid gradient sweep). Use CSS keyframes, not Tailwind `animate-pulse` — consistent with stepFadeUp pattern in globals.css.
- Skeleton bar dimensions: `width: 80px`, `height: 20px` (matches price text slot).
- No spinner. No dash placeholder. Skeleton prevents layout shift.

### PriceSummary Panel

- Desktop: sticky right column within the step layout. `position: sticky`, `top: 24px`.
- Mobile: fixed bottom bar. `position: fixed`, `bottom: 0`, `left: 0`, `right: 0`, `height: 56px`, `background: var(--anthracite-mid)`, `border-top: 1px solid var(--anthracite-light)`.
- Desktop content: `.label` "YOUR JOURNEY" + origin → destination (truncated at 28 chars with ellipsis) + selected vehicle class name (or "—" if none selected) + base price (or "—").
- Mobile content: total price on the left + "Next" `.btn-primary` on the right, same row.
- Transition: price number cross-fades on vehicle switch (opacity 0→1, 150ms).

## Copywriting Contract

All copy follows STYLEGUIDE brand voice: confident, warm, discreet. No hype. No hedging.

| Element | Copy |
|---------|------|
| Step 2 heading | Select your date & time |
| Step 2 date label | PICKUP DATE |
| Step 2 return date label | RETURN DATE |
| Step 2 time label | PICKUP TIME |
| Step 3 heading | Choose your vehicle |
| Primary CTA (Step 2 → 3) | Continue |
| Primary CTA (Step 3 → 4) | Continue |
| Back button | Back |
| Price loading state | (skeleton only — no text) |
| Quote fallback on card | Request a quote |
| Empty state — no date selected | Select a pickup date to continue |
| Empty state — no vehicle selected | Select a vehicle to continue |
| Error state — price fetch failed | Pricing unavailable. Your selection has been saved — continue to request a quote. |
| PriceSummary — no selection | — (em dash, no placeholder text) |
| PriceSummary label | YOUR JOURNEY |
| Passenger capacity label | PASSENGERS |
| Luggage capacity label | LUGGAGE |

Destructive actions in Phase 2: none.

Voice notes:
- "Continue" not "Next" — warmer, less transactional.
- "Select a vehicle to continue" not "Please choose a vehicle" — declarative, not pleading.
- "Request a quote" not "Get a quote" or "Contact us" — service-first framing.

## Animation Contract

All new animations must be CSS keyframes in `globals.css` — no framer-motion (source: WIZD-05).

| Animation | Trigger | Implementation |
|-----------|---------|----------------|
| Step entry | Step becomes active | `.animate-step-enter` — `stepFadeUp` 0.3s (already in globals.css) |
| Skeleton shimmer | Price loading | New `@keyframes shimmer` in globals.css — copper-pale sweep |
| Price cross-fade | Vehicle card switch | `opacity: 0` → `opacity: 1`, 150ms ease, in-place |
| Vehicle card selection | User clicks card | `border-color 0.2s ease, background-color 0.2s ease` |

Reduced motion: all animations must respect `@media (prefers-reduced-motion: reduce)` — the globals.css block already covers `.animate-step-enter`. Shimmer must also be suppressed.
