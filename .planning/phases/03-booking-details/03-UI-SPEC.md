# Phase 3 — UI Design Contract

> Visual and interaction contract for Phase 3: booking-details (Steps 4 & 5).
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — hand-rolled design system |
| Preset | not applicable |
| Component library | none (custom components only) |
| Icon library | none (text labels and unicode only, per existing pattern) |
| Font (display) | Cormorant Garamond — weights 300, 400, 600 |
| Font (body/UI) | Montserrat — weights 200, 300, 400, 500, 600 |

Source: `STYLEGUIDE.md`, `prestigo/app/globals.css`

All design tokens live in CSS custom properties declared in `:root` in `globals.css`.
Never use hex values directly in component inline styles — always reference `var(--token-name)`.
Exception: `#C0392B` for validation error states (established pattern from `AddressInput.tsx`).

## Spacing Scale

Declared values (multiples of 4 where possible; brand guide uses 6/12/24/48/80 — mapped to nearest standard):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tight internal spacing |
| sm | 8px | Label-to-field margin, char counter margin-top |
| md | 16px | Input padding horizontal, field-to-field gap (mobile) |
| lg | 24px | Field row gaps (desktop), card padding, section gaps |
| xl | 32px | Button horizontal padding |
| 2xl | 48px | Step container top padding |
| 3xl | 64px | Page-level section breaks |

Exceptions:
- Input padding vertical: 12px (established `AddressInput.tsx` pattern — `padding: '12px 16px'`)
- Extra toggle card padding: 23px when selected (2px border), 24px when unselected (1px border) — established `VehicleCard.tsx` compensation pattern
- Mobile sticky bar height: 56px fixed (PriceSummary mobile bar — established Phase 2 pattern)
- Touch targets (buttons): minimum 44px height on mobile
- Label margin-bottom: 8px before input fields (established `AddressInput.tsx` pattern)

Source: `STYLEGUIDE.md` spacing table, `prestigo/app/globals.css`, `VehicleCard.tsx` border compensation pattern

## Typography

Phase 3 uses four type roles. All text elements must use `var(--font-montserrat)` or `var(--font-cormorant)` — never a raw font-family string.

| Role | Font | Size | Weight | Line Height | Letter Spacing | Color |
|------|------|------|--------|-------------|----------------|-------|
| Step heading (h2) | Cormorant Garamond | 26px | 300 | 1.25 | 0 | `var(--offwhite)` |
| Body / input value | Montserrat | 14px | 300 | 1.8 | 0.03em | `var(--offwhite)` |
| Label (section, field) | Montserrat | 9px | 400 | — | 0.4em (uppercase) | `var(--copper)` |
| Caption / meta | Montserrat | 10px | 300 | — | 0.03em | `var(--warmgrey)` |

Additional rules:
- All `.label` elements: `text-transform: uppercase`, `letter-spacing: 0.4em` — use the `.label` CSS class
- Price display (PriceSummary): 20px / weight 500 / `var(--offwhite)` when vehicle selected; 13px / weight 400 / `var(--warmgrey)` when no selection
- Error messages: 13px / weight 400 / `#C0392B`, `margin-top: 8px`
- Character counter (textarea): 10px / weight 300 / `var(--warmgrey)`, right-aligned
- Extra price increment label: 13px / weight 300 / `var(--warmgrey)` — e.g. "+€15"
- CTA button text: 10px / weight 400 / letter-spacing 0.35em / uppercase — use `.btn-primary` CSS class
- Never use bold or heavy weights for headings — lightness signals luxury (STYLEGUIDE rule)

Source: `STYLEGUIDE.md` type scale, `prestigo/app/globals.css`, `AddressInput.tsx` input pattern, `PriceSummary.tsx` price display

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#1C1C1E` (`var(--anthracite)`) | Page background, wizard container background |
| Secondary (30%) | `#2A2A2D` (`var(--anthracite-mid)`) | Extra toggle cards, input field backgrounds, PriceSummary panel, mobile sticky bar |
| Accent (10%) | `#B87333` (`var(--copper)`) | See reserved list below |
| Destructive | `#C0392B` | Inline validation error text and input border only |

Accent (`var(--copper)`) reserved for:
1. Selected extra toggle card border (2px solid — selected state)
2. Step label text ("STEP 4 OF 6", "STEP 5 OF 6") via `.label` class
3. Copper accent line (`<span className="copper-line">`) above step heading
4. Continue button hover background (`.btn-primary:hover`)
5. Focus-visible outline on all interactive elements (`:focus-visible` — 2px solid, offset 4px)
6. Field label color is `var(--copper)` via `.label` class

Borders:
- Unselected extra toggle card: `1px solid var(--anthracite-light)` (`#3A3A3F`)
- Input fields default border: `1px solid var(--anthracite-light)`
- Input fields error border: `1px solid #C0392B`
- Dividers: `1px solid var(--anthracite-light)`
- PriceSummary panel border: `1px solid var(--anthracite-light)`

Source: `STYLEGUIDE.md` colour palette, `prestigo/app/globals.css` tokens, `VehicleCard.tsx` selected/unselected border pattern, `AddressInput.tsx` error pattern

## Component Inventory

### Step 4 — Extras (new: `Step4Extras.tsx`)

Three toggle cards in a vertical list (mobile) or 1–3 column grid (desktop, if space allows — default vertical).

Each extra toggle card:
- `<button type="button" aria-pressed={boolean}>` — accessible toggle, never a checkbox
- Selected state: `border: 2px solid var(--copper)`, `padding: 23px` (compensates border width), `background: var(--anthracite-mid)`
- Unselected state: `border: 1px solid var(--anthracite-light)`, `padding: 24px`, `background: var(--anthracite-mid)`
- `border-radius: 4px`
- Content layout: extra name (body 14px / `var(--offwhite)`) + price increment (13px / `var(--warmgrey)`) + optional description line (10px / `var(--warmgrey)`)
- Transition: `transition: border-color 0.2s ease, padding 0.0s` (instant padding to avoid jank)

Extras defined:
| Key | Display Name | Price | Description |
|-----|-------------|-------|-------------|
| `childSeat` | Child Seat | +€15 (placeholder) | Safety seat for children up to 36 kg |
| `meetAndGreet` | Meet & Greet | +€25 (placeholder) | Name board at arrivals |
| `extraLuggage` | Extra Luggage | +€20 (placeholder) | Additional luggage allowance |

Step 4 is always valid (all extras optional) — Continue is always enabled.

### Step 5 — Passenger Details (new: `Step5Passenger.tsx`)

Form uses `react-hook-form` with `mode: 'onBlur'` and Zod 4 schema resolver.

Field layout:
- Row 1: First Name + Last Name — side by side on desktop (`flex-row gap-24px`), stacked on mobile (`flex-col gap-16px`)
- Row 2: Email — full width
- Row 3: Phone — full width
- Row 4 (airport rides only): Flight Number (required) + Terminal (optional) — side by side on desktop, stacked on mobile
- Row 5: Special Requests / Notes — full-width textarea, 4 rows, 500 char max with live counter

Each field structure:
```
<p className="label" style={{ marginBottom: '8px' }}>FIELD NAME</p>
<input style={{ background: var(--anthracite-mid), border: 1px solid [normal|error], padding: 12px 16px, ... }} />
<p style={{ color: '#C0392B', fontSize: 13, marginTop: 8 }}>{error message}</p>  {/* only when error */}
```

Validation errors appear on blur only (not on submit, not on keystroke). Source: REQUIREMENTS.md STEP5-04.

Airport detection: `origin?.placeId === PRG_CONFIG.placeId || destination?.placeId === PRG_CONFIG.placeId` — renders Flight Number field conditionally.

Textarea character counter: right-aligned below textarea, `{n}/500` in 10px / `var(--warmgrey)`.

### PriceSummary — Extras Extension

PriceSummary reads `extras` from Zustand store and computes `extrasTotal` locally using `EXTRAS_PRICES` constant from `lib/extras.ts` (new client-safe module). Display total = `selectedPrice.base + extrasTotal`.

At Step 4+: PriceSummary mobile bar Continue button hidden (reads `currentStep` from store — only visible at Step 3). Wizard shell generic mobile bar takes over for Steps 4 and 5.

Desktop panel extension: after price line, show extras breakdown if any are selected:
- Each selected extra: `{Extra Name} +€{price}` in 13px / `var(--warmgrey)`
- Total line: `var(--offwhite)` with 20px / weight 500

### BookingWizard — Step Headings Extension

Steps 4 and 5 use the same heading block as steps 1–3:
- Section label: `STEP {N} OF 6` via `.label` class
- Copper line: `<span className="copper-line mb-6 block" />`
- h2: Cormorant Garamond 26px / weight 300 / `var(--offwhite)` / line-height 1.25

Step 4 heading text: "Add extras"
Step 5 heading text: "Passenger details"

Source: `BookingWizard.tsx` heading block pattern, `03-RESEARCH.md` BookingWizard step headings extension

## Responsive Behavior

| Breakpoint | Layout |
|-----------|--------|
| Mobile (<768px `md` breakpoint) | Single column. Fields stacked. Extra cards full width. Mobile sticky bar (wizard shell) shows Back + Continue. PriceSummary mobile bar shows price only (no Continue at steps 4–5). |
| Desktop (≥768px) | Two-column wizard layout: form content (left) + PriceSummary sticky panel (right). Name fields side-by-side. Flight Number + Terminal side-by-side. |

Mobile sticky bar (wizard shell) height: 72px fixed, `bottom: 0`, `z-index: 50`.
PriceSummary mobile bar height: 56px. At steps 4 and 5, PriceSummary mobile bar hides its Continue button (currentStep !== 3 guard).

Source: Phase 2 established dual-bar pattern, `BookingWizard.tsx`, `PriceSummary.tsx`

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| Extra toggle cards | `<button type="button" aria-pressed={boolean}>` |
| Input error association | `aria-describedby` pointing to error `<p>` element id |
| Required fields | `aria-required="true"` on First Name, Last Name, Email, Phone, Flight Number (when airport) |
| Flight Number conditional | `aria-required` toggled based on `isAirportRide` |
| Focus order | DOM order matches visual order — no `tabIndex` manipulation |
| Focus ring | `:focus-visible` 2px copper, offset 4px (global `globals.css` rule) |
| Stepper aria | `aria-disabled` (not native `disabled`) — established Phase 1 pattern |

Source: Phase 1/2 accessibility decisions in `STATE.md`, `globals.css` `:focus-visible` rule
