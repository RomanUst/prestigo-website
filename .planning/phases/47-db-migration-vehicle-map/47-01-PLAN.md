<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| anon/public Supabase client → gnet_bookings | Untrusted callers must not be able to read or write GNet booking data |
| bookings DELETE → gnet_bookings child rows | Cascading delete would silently destroy GNet audit trail |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-47-01 | Tampering | gnet_bookings table via anon Supabase key | mitigate | RLS policies `USING (false)` + `WITH CHECK (false)` for SELECT/INSERT/UPDATE/DELETE — anon role cannot interact with table; only service_role bypasses RLS |
| T-47-02 | Tampering | Duplicate transaction_id race condition (two Farm In requests with same tx) | mitigate | DB-level UNIQUE constraint on transaction_id rejects second INSERT with 23505 — Phase 49 catches this for idempotency |
| T-47-03 | Repudiation | gnet_bookings row deleted when parent bookings row is removed (audit trail loss) | mitigate | FK ON DELETE RESTRICT — bookings row cannot be deleted while gnet_bookings child exists |
| T-47-04 | Tampering | Orphan gnet_bookings without matching bookings row | mitigate | booking_id NOT NULL + FK constraint at DB level |
| T-47-05 | Information Disclosure | raw_payload JSONB may contain partner-sensitive fields | accept | Service-role only access via RLS; no direct client exposure; Phase 51 admin UI displays summary fields, not raw payload |
</threat_model>
