  <action>
Verify the server-side Google Maps API key restriction in Google Cloud Console. This key is used by `prestigo/app/api/calculate-price/route.ts` for server-to-server Routes API calls. Vercel serverless functions do NOT send a Referer header, so any HTTP referrer restriction will cause REQUEST_DENIED on all pricing API calls.

Steps:
1. Go to Google Cloud Console -> APIs and Services -> Credentials
2. Find the API key used as `GOOGLE_MAPS_API_KEY` (the server-side key)
3. Click "Edit" on that key
4. Check "Application restrictions" section:
   - CORRECT states: "None" or "IP addresses" — leave as-is
   - WRONG state: "HTTP referrers (websites)" — change to "None" and click Save
5. Check "API restrictions" section:
   - Should be restricted to Routes API (or Distance Matrix API) only
   - If "Don't restrict key", consider adding API restriction for security (optional)

Report: the current Application restrictions value, and whether any change was made.
  </action>
  <verify>User confirms Application restrictions = "None" or "IP addresses" (not "HTTP referrers")</verify>
  <acceptance_criteria>

    - Application restrictions is NOT set to "HTTP referrers (websites)"
  </acceptance_criteria>

2. Find the API key used as `NEXT_PUBLIC_GOOGLE_MAPS_API_KEY` (the client-side key)
3. Click "Edit" on that key
4. Check "Application restrictions" section:
   - Should be "HTTP referrers (websites)"

   - Set Application restrictions to "HTTP referrers (websites)"
   - Add item: `https://rideprestige.com/*`
   - Optionally also add `https://www.rideprestige.com/*` if www subdomain is used
   - Click Save
6. Verify API restrictions include: Maps JavaScript API, Places API (or Places API (New))

Report: the current referrer list, and whether any change was made.
