## Interaction Contracts

### Widget CTA — "Book Now"

1. User fills: trip type + origin + destination (or duration for Hourly) + date + time
2. User clicks "Book Now"
3. Widget calls all Zustand setters: `setTripType`, `setOrigin`, `setDestination` / `setHours`, `setPickupDate`, `setPickupTime`
4. Sets `currentStep: 3` in store
5. Adds steps 1 and 2 to `completedSteps` Set so progress bar shows them as done
6. Calls `router.push('/book')`
7. User lands at `/book` already on Step 3 (vehicle selection) with prices visible

### Airport trip type auto-fill behavior

When widget trip type is Airport Pickup or Airport Dropoff:
- Origin (Airport Pickup) or Destination (Airport Dropoff) auto-fills to `PRG_CONFIG` coordinates
- Same behavior as Step 1 in BookingWizard

### Date/time min constraints (Claude's Discretion — resolved here)

- `<input type="date">` min attribute: set to today's date (computed client-side on mount via `new Date().toISOString().split('T')[0]`)
- `<input type="time">` min attribute: only enforce when selected date equals today; otherwise no minimum

### Mobile keyboard visibility (UX-03)

- On `focus` of any text input or address field in the widget or wizard, call `scrollIntoView({ behavior: 'smooth', block: 'center' })` on the input element
- CTA buttons (Next, Continue, Pay) must remain in viewport above the virtual keyboard — use `position: sticky; bottom: 0` with `padding-bottom: env(safe-area-inset-bottom)` where needed

### Tab order (UX-05)

- Use DOM order for tab sequence — do NOT use positive `tabIndex` values
- `tabIndex={0}` only to make non-focusable elements (e.g. custom divs acting as buttons) keyboard reachable
- `tabIndex={-1}` for elements that are reachable programmatically but should not be in the tab ring

### aria-label text for icon-only buttons (UX-04)

| Button | aria-label |
|--------|-----------|
| Swap origin/destination | "Swap origin and destination" |
| Stepper increment (+) | "Increase {context, e.g. passengers}" |
| Stepper decrement (-) | "Decrease {context, e.g. passengers}" |
| Clear/close address field (×) | "Clear address" |
| Close modal or overlay | "Close" |
