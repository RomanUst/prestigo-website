## Spacing Scale

Declared values (multiples of 4). Derived from observed class usage across existing article and listing pages.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline gaps between byline elements, bullet dot margin-top |
| sm | 8px | Compact stack gaps (e.g. `gap-2` between list items) |
| md | 16px | Default horizontal/vertical element spacing inside cards |
| lg | 24px | Section inner padding, gap between card sections |
| xl | 32px | Horizontal card padding on desktop (`p-8` = 32px) |
| 2xl | 48px | Section vertical padding mobile (`py-12`) |
| 3xl | 64px | Major section vertical padding desktop (`py-16` = 64px, `py-24` = 96px) |

Exceptions:
- Touch targets: all interactive elements (`<a>`, `<button>`) must have `min-height: 44px` (matches `.btn-primary` rule ŌĆö WCAG 2.5.5 / Apple HIG).
- Card hover border transitions: `transition-colors` only, no layout shift.
- Hero section top padding: `pt-32` mobile (128px), `pt-40` desktop (160px) ŌĆö accounts for fixed Nav height.

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Heading hierarchy | h1 on listing hero, h2 on each card title, h2 on section headings in article |
| Link text | Cards are `<a>` wrapping entire card ŌĆö add `aria-label="{title}"` to avoid "Read article ŌåÆ" as sole link text |
| Cover image alt | `alt={post.title}` on listing cards; `alt={post.title}` on article hero (decorative images exempt) |
| Touch targets | `min-height: 44px` on all `<a>` and `<button>` (inherited from `.btn-primary` pattern) |
| Skip link | Existing `.skip-link` targets `#main-content` ŌĆö `<main id="main-content">` required on both pages |
| Focus visible | Global `:focus-visible` rule provides 2px copper outline |
| WCAG contrast | Category labels use `--copper-light` (#D4924A) not `--copper` (#B87333) on `--anthracite-mid` backgrounds (5.6:1 ratio vs 3.77:1 ŌĆö AA compliant) |
