<objective>
Extend `/api/calculate-price` to accept `returnTime` in the request body and compute the return-leg price independently: apply night/holiday coefficients derived from `returnDate + returnTime` (not from pickup), apply `returnDiscountPercent`, and NEVER add extras. Response shape adds a new `returnLegPrices` field (per-vehicle return-leg price only); the existing combined-total `roundTripPrices` field is replaced by `returnLegPrices`.

Purpose: Phase 25 Requirements RTPR-01 (independent coefficient from return datetime), RTPR-02 (discount applied, % matches), RTPR-03 (no extras on return). The current API multiplies outbound total by 2 and applies discount — it inherits outbound night/holiday coefficients incorrectly for return-leg pricing.
Output: API returns `{ prices, returnLegPrices, returnDiscountPercent, distanceKm, quoteMode }` where `returnLegPrices[vc].total` is the return-leg adjusted + discounted price.
</objective>

       let returnLegPrices: Record<string, { base: number; extras: number; total: number; currency: string }> | null = null

             return [vc, { base: discountedTotal, extras: 0, total: discountedTotal, currency: b.currency }]
