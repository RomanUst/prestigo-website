## Spacing Scale

Declared values (existing project tokens, confirmed in STYLEGUIDE.md § 6 and component inline styles):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, status dot, badge padding |
| sm | 8px | Input padding, badge padding, list item gap |
| md | 16px | Nav item vertical rhythm, card internal padding |
| lg | 24px | Sidebar header padding, section padding |
| xl | 32px | Main content padding (desktop), layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level top/bottom spacing |

Exceptions:
- Touch targets: all `<button>` and `<Link>` interactive elements on mobile must have `minHeight: 44px` and `minWidth: 44px` (UX-01 requirement). This overrides the standard sm/md padding on nav links and icon buttons.
- Sidebar nav link height on mobile: `minHeight: 44px` (expanded from current 10px vertical padding + 13px font).
- Promo input field: `padding: 10px 12px` (48px total height) to match the 44px accessibility floor.

Source: REQUIREMENTS.md UX-01, RESEARCH.md Pattern 5, STYLEGUIDE.md § 6 Spacing

## Interaction Contracts

### Mobile Sidebar (UX-01)

| State | Visual |
|-------|--------|
| Mobile — closed | Hamburger `<button>` (44x44px) fixed top-left; `<aside>` hidden (`display: none` via CSS `hidden md:flex` class) |
| Mobile — open | `<aside>` position: fixed, top: 0, left: 0, width: 280px, height: 100vh, z-index: 50, background: `#1C1C1E`; backdrop overlay `div` with `rgba(0,0,0,0.5)` covers the rest |
| Desktop (≥768px) | Hamburger hidden (`md:hidden`); `<aside>` always visible, width: 240px, position: static, no overlay |
| Nav link active | 3px solid `#B87333` left border; text color `#F5F2EE`; `minHeight: 44px` |
| Nav link inactive | 3px transparent left border; text color `#9A958F`; `minHeight: 44px` |
| Sign out button | `minHeight: 44px`, `minWidth: 44px` — touch-safe |

### BookingsTable Mobile Card Layout (UX-01)

| State | Visual |
|-------|--------|
| Desktop (≥768px) | Standard `<table>` layout — no change from current implementation |
| Mobile (<768px) | Hide `<table>` via `hidden md:block` CSS class; render stack of `<div>` booking cards |
| Mobile card anatomy | booking_reference (13px, copper if manual badge), client name (13px, offwhite), status badge (StatusBadge component), pickup date + time (11px, warmgrey), vehicle class (11px, warmgrey), CZK amount (13px, offwhite, right-aligned), action buttons (44px touch height) |
| Expanded mobile card | Tap card to expand; shows all detail fields in 2-column grid (same DetailField component reused); operator notes textarea stays as-is |

### PromoCodesTable (PROMO-01, PROMO-02)

| State | Visual |
|-------|--------|
| Table columns | CODE, DISCOUNT, EXPIRY, USES, STATUS, ACTIONS |
| Active promo row | StatusBadge variant "active" (green); row background `#2A2A2D` |
| Inactive promo row | StatusBadge variant "inactive" (red); row background `#1C1C1E`; text color `#9A958F` (dimmed) |
| Deactivate toggle | Lucide `ToggleLeft` (active) / `ToggleRight` (inactive) icon button; 44px touch target; clicking toggles `is_active` via PATCH — optimistic update |
| Delete action | Lucide `Trash2` icon button; color `#9A958F` default, `#f87171` on hover; 44px touch target; inline confirmation: button text changes to "CONFIRM?" for 3s then reverts — no modal needed (single-operator context) |
| Empty state | Heading: "No promo codes yet." Body: "Create a code to offer a discount at checkout." Action: "Create Code" button (copper CTA) |
| Loading state | Row cells replaced with a single `…` skeleton-style span at 40% opacity |

### PromoCodeForm (PROMO-01)

| Field | Validation | Visual |
|-------|------------|--------|
| CODE | 3–50 chars, auto-uppercased | Input width: 160px; placeholder "SUMMER20"; shows current value uppercased |
| DISCOUNT % | 1–100, number | Input width: 80px; suffix label "%" right of input |
| EXPIRY DATE | ISO date or empty (no expiry) | Date input; placeholder "No expiry"; empty = unlimited |
| USAGE LIMIT | Positive integer or empty (unlimited) | Input width: 80px; placeholder "Unlimited"; empty = null |
| Submit | POST /api/admin/promo-codes | CTA label: "Add Code"; copper button; disabled during submit; shows spinner icon while loading |
| Duplicate code error | 400 from server | Inline red error below CODE field: "Code already exists." |
| Success | Code appears in table | Form resets; new code added optimistically to top of table |

### PromoInput in Booking Wizard (PROMO-03)

Location: `Step6Payment.tsx`, inserted between BookingSummaryBlock and the Stripe PaymentElement.

| State | Visual |
|-------|--------|
| Default | Label "PROMO CODE" (11px, uppercase, warmgrey); text input (full width, anthracite-mid bg, anthracite-light border, offwhite text); "Apply" button (copper, 44px height, 13px Montserrat) |
| Loading | "Apply" button shows "…" text, disabled |
| Valid code applied | Green checkmark (Lucide `CheckCircle2`, #4ade80) inline with code; below summary line: "PROMO: −{X}%" in green (`#4ade80`); total line updates to show discounted amount; "Remove" link (`#9A958F`, 11px) to clear the code |
| Invalid code | Input border changes to `#f87171`; error text below: specific message from server (e.g. "Code not found, expired, or inactive." / "This promo code has reached its usage limit.") |
| Concurrent exhaustion at payment | Payment error dismisses and shows: "Your promo code was just fully redeemed. Please proceed without it or contact us." (no inline color change — uses existing payment error display) |
| Code cleared | Discount line removed; total reverts to base price; input cleared |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA — create promo | "Add Code" |
| Primary CTA — promo in wizard | "Apply" |
| Primary CTA — main admin nav | "Promos" (nav label, uppercase) |
| Empty state heading — promo table | "No promo codes yet." |
| Empty state body — promo table | "Create a code to offer a discount at checkout." |
| Empty state action — promo table | "Create Code" |
| Error — duplicate code | "Code already exists." |
| Error — code not found / expired | "Code not found, expired, or inactive." |
| Error — code exhausted | "This promo code has reached its usage limit." |
| Error — concurrent exhaustion at payment | "Your promo code was just fully redeemed. Please proceed without it or contact us." |
| Error — generic server error | "Something went wrong. Please try again." |
| Discount applied label | "PROMO: −{X}%" |
| Deactivate confirmation | Inline: button becomes "CONFIRM?" for 3 seconds — no modal |
| Delete confirmation | Inline: button becomes "CONFIRM?" for 3 seconds — no modal |
| Deactivated badge label | "INACTIVE" |
| Active badge label | "ACTIVE" |
| Unlimited expiry label | "No expiry" |
| Unlimited uses label | "Unlimited" |

Notes:
- Do NOT use "Amazing deal", "Save now", "Limited time" — inconsistent with PRESTIGO luxury voice (STYLEGUIDE.md § 5)
- Promo codes in admin are business tools, not marketing copy — tone is factual and operational
- "Remove" not "Delete" for clearing applied promo in wizard — softer action language for clients
- "Deactivate" not "Disable" — more professional for operator context

Source: STYLEGUIDE.md § 5 (brand voice), REQUIREMENTS.md PROMO-01/02/03/04
