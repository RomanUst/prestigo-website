### Manager Alert Email (plain-text)

| Element | Copy |
|---------|------|
| Email subject | `New booking: {bookingReference} — {clientFirstName} {clientLastName}` |
| Body preamble | `New booking received. Details below.` |
| Fields order | Booking reference, Client name, Email, Phone, Route, Vehicle class, Pickup date + time, Flight number (if airport ride), Extras, Total (CZK) |

Source: CONTEXT.md

### Emergency Fallback Email (Supabase failure)

| Element | Copy |
|---------|------|
| Email subject | `ALERT: Supabase save failed — {bookingReference}` |
| Body | `Supabase save failed after 3 retries. Full booking data as JSON follows. Client confirmation email has been sent.` |

Source: CONTEXT.md (failure handling section)

### No-UI States (backend only — no user-facing copy needed)

| Scenario | Handling |
|----------|---------|
| Webhook returns 200 on partial failure | No user-facing message — Stripe must not retry |
| Quote mode (no client confirmation email) | Manager alert sent; client receives no email (not confirmed yet) |
| Duplicate webhook event | Deduplicate by `paymentIntentId` in Supabase — silent no-op, return 200 |

### Destructive Actions

None in Phase 5. No destructive UI actions are implemented this phase.

## Implementation Notes for Executor

1. **No new page routes or React components** — Phase 5 is server-side only.
2. **Email as template string** is the simplest path. React Email (`@react-email/components`) is permitted if Claude prefers it — it produces inline-styled HTML compatible with email clients.
3. **Inline styles mandatory** in all email HTML — no class attributes, no external CSS.
4. **Google Fonts in email**: include `<link>` to load Montserrat in the `<head>` for clients that support it; Arial fallback handles all others.
5. **Booking reference format**: `PRG-YYYYMMDD-NNNN` for paid bookings, `QR-YYYYMMDD-NNNN` for quote mode — already generated in Phase 4 stub.
6. **EUR display**: use `czkToEur()` from `prestigo/lib/currency.ts` — already implemented.
7. **Extras labels**: use `EXTRAS_CONFIG` from `prestigo/lib/extras.ts` for human-readable names.
