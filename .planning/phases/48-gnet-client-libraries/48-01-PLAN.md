<objective>
Build CLIENT-01: a server-only GNet auth token cache that mirrors the lib/flight-cache.ts Upstash REST pattern, falls back to in-memory caching when Upstash env vars are absent, and exposes typed GnetTokenError values so downstream callers (CLIENT-02 in plan 02) can implement retry-on-401 cleanly.

Purpose: GNet's getToken2 issues JWTs with a 3600s TTL. Calling auth on every outbound request is wasteful and slow; persisting the JWT in Upstash with a 3500s TTL (100s safety margin) gives every Vercel function instance a shared cache. In-memory fallback keeps local dev and unconfigured environments working without code changes.

Output: lib/gnet-token.ts (production module) + tests/gnet-token.test.ts (7-test vitest suite, all green).
</objective>

<!-- Reference pattern this module mirrors. -->

From lib/flight-cache.ts (Upstash REST pattern):
- Direct fetch to ${UPSTASH_REDIS_REST_URL}/get/<key> with Bearer auth
- POST to base URL with body ['SET', key, value, 'EX', ttl]
- cache: 'no-store' on every request
- Silent failure (return null/false) when env vars are unset

Public surface this plan exports:
```typescript
export type GnetTokenErrorCode =
  | 'MISSING_CREDENTIALS'
  | 'AUTH_FAILED'
  | 'NETWORK_ERROR'
  | 'PARSE_ERROR'

export class GnetTokenError extends Error {
  readonly code: GnetTokenErrorCode
  readonly cause?: unknown
}

export function getGnetToken(force?: boolean): Promise<string>
```

Env vars consumed:
- GNET_UID, GNET_PW (required) — credentials for getToken2
- GNET_AUTH_URL (optional) — defaults to https://api.grdd.net/Platform.svc/getToken2
- UPSTASH_REDIS_REST_URL, UPSTASH_REDIS_REST_TOKEN (optional) — falls back to memory if unset

Each env read calls .replace(/\\n$/, '').trim() to strip Vercel-style trailing-newline pollution.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Prestigo server → GNet auth API | Outbound credentials cross to a third-party service; response (JWT) re-enters our trust boundary |
| Prestigo server → Upstash Redis REST | Token at rest in third-party cache; protected by Upstash Bearer token only |
| lib/gnet-token.ts → logging sinks (Vercel logs) | Secrets must never reach stdout/stderr |

## STRIDE Threat Register (ASVS L1)

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-48-01 | Information Disclosure | console.{log,warn,info} in lib/gnet-token.ts | mitigate | Test 7 spies on all three console methods and asserts the raw token never appears; only the literal string '[gnet-token] token refreshed' is logged on refresh |
| T-48-02 | Information Disclosure | GnetTokenError.message on AUTH_FAILED | mitigate | Test 5 asserts the message must NOT contain GNET_PW value; implementation uses only the HTTP status code in the message |
| T-48-03 | Spoofing | Missing/invalid GNET_UID/GNET_PW silently calling auth | mitigate | fetchNewToken throws MISSING_CREDENTIALS before any network call when uid/pw are falsy after trim |
| T-48-04 | Tampering | Trailing-newline pollution in env vars (Vercel paste artefact) | mitigate | Every `process.env.GNET_*` read is followed by `.replace(/\\n$/, '').trim()` |
| T-48-05 | DoS | Token thrash if cache TTL ≥ GNet TTL | mitigate | TOKEN_TTL = 3500s with 100s safety margin under GNet's 3600s; force-refresh path exists for retry-on-401 (consumed in plan 02) |
| T-48-06 | Repudiation | No audit trail when refresh happens | accept | Single `console.info('[gnet-token] token refreshed')` provides minimum signal; full audit deferred — no PII involved |
| T-48-07 | Elevation of Privilege | Module imported into Edge runtime / client bundle leaking secrets | accept | Module is server-only by construction (reads process.env). No 'use client' directive. Existing Next.js boundary discipline keeps it out of client bundles |

**Block-on:** T-48-01 and T-48-02 are mitigated by Test 5 and Test 7. The plan does not ship until both pass.
</threat_model>
